组件的生命周期钩子

  • 2022-01-25
  • 作者 西瓜
  • 636 字
本文最后编辑于 前，其中的内容可能需要更新。

从组件渲染 —–> 销毁。不同的生命周期阶段对应着组件内不同的生命周期函数。在对应阶段，对应的函数会自动触发。所有的声明周期都是同步函数。

1. 初始渲染阶段

① beforeCreate
在实例初始化之后，数据观测 (data observer) 和 event/watcher 事件配置之前被调用。(还没有渲染页面，在即将渲染页面之前)

② created
data 设置完成，可以修改了，可以执行修改 data 的操作，也就是以后的异步请求修改数据可以在此生命周期执行。

③ beforeMount
在挂载开始之前被调用：相关的 render 函数首次被调用。(挂载指的是渲染) 该钩子在服务器端渲染期间不被调用。

④ mounted
组件完全渲染到了页面中，可以获取真实的 dom 节点，也可以发送请求更新数据。(在 mounted 和 created 发送异步请求更新数据是一样的，没有任何区别)

2. 数据更新渲染 (当 data prop 发生改变的时候)

① beforeUpdate
数据更新时
② updated
数据更新完毕之后，页面更新也完毕了，可以获取更新之后的 data 以及真实的 dom 节点。

3. 销毁

① beforeDestroy
实例(组件)销毁之前调用，在这一步，实例仍然完全可用。
② destroyed
对应 Vue 实例的所有指令都被解绑，所有的事件监听器被移除，所有的子实例也都被销毁。比如清除计数器，清除 window 的滚动条事件等。
